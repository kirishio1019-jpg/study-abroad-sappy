# ビルド成功後の404エラー - 実行時エラーの確認方法

## 🔍 確認手順

ビルドが成功している場合、実行時のエラーが原因の可能性が高いです。

### ステップ1: Function Logsを確認

1. **Vercelダッシュボードにアクセス**
   - https://vercel.com/dashboard
   - プロジェクト「study-abroad-sappy」を開く
   - 最新のデプロイを開く

2. **「Function Logs」タブを確認**
   - 実行時のエラーメッセージがあるか確認
   - エラーメッセージがあれば、その内容を全てコピーしてください

### ステップ2: ブラウザのコンソールを確認

1. **デプロイURLにアクセス**
   - VercelダッシュボードでデプロイURLを確認
   - そのURLにアクセス

2. **開発者ツールを開く**
   - F12キーを押す
   - または、右クリック → 「検証」

3. **「Console」タブを確認**
   - エラーメッセージがあるか確認
   - エラーメッセージがあれば、その内容をコピーしてください

### ステップ3: 環境変数の確認

**Settings → Environment Variables** で以下が設定されているか確認：

- ✅ `NEXT_PUBLIC_SUPABASE_URL`
- ✅ `NEXT_PUBLIC_SUPABASE_ANON_KEY`
- ✅ すべての環境（Production, Preview, Development）に設定されているか

## 🚨 よくあるエラー

### エラー1: 環境変数が設定されていない

**症状**:
- Function Logsに環境変数関連のエラーが表示される

**解決方法**:
- Settings → Environment Variables で設定を確認
- すべての環境に設定されているか確認

### エラー2: Supabaseの接続エラー

**症状**:
- Function LogsにSupabase関連のエラーが表示される

**解決方法**:
- 環境変数が正しく設定されているか確認
- Supabaseのプロジェクトが正しく設定されているか確認

### エラー3: ルーティングの問題

**症状**:
- 404エラーが特定のページでのみ発生する

**解決方法**:
- そのページのファイルが正しく存在するか確認
- `app/page.tsx`が正しくエクスポートされているか確認

## 📋 確認してほしい情報

以下の情報を共有してください：

1. **Function Logsにエラーがあるか**
   - あれば、エラーメッセージを全てコピー

2. **ブラウザのコンソールにエラーがあるか**
   - あれば、エラーメッセージを全てコピー

3. **どのページで404エラーが表示されるか**
   - ルートページ（`/`）か、それ以外か

4. **環境変数が設定されているか**
   - Settings → Environment Variables で確認

この情報があれば、具体的な解決方法を提案できます。
