# 🔴 「このサイトにアクセスできません」エラーの解決方法

## エラーの原因

ブラウザで `http://localhost:3000` にアクセスしたときに「このサイトにアクセスできません」と表示される場合、以下のいずれかが原因です：

1. **開発サーバーが起動していない**
2. **開発サーバーが別のポートで起動している**
3. **開発サーバーでエラーが発生している**

---

## ✅ 解決方法

### ステップ1: 開発サーバーが起動しているか確認

#### 確認方法

開発サーバーを起動した**コマンドプロンプトのウィンドウ**（またはターミナル）を確認してください。

以下のようなメッセージが表示されていれば、サーバーは起動しています：
```
✓ Ready in X seconds
○ Local:        http://localhost:3000
```

#### もしこのメッセージが表示されていない場合

→ **開発サーバーを起動する必要があります**

##### 起動方法：

1. **コマンドプロンプトを開く**
   - Windowsキーを押す
   - 「cmd」と入力してEnter

2. **以下のコマンドを実行：**
   ```
   cd C:\Users\kiris\.cursor\study-abroad-comparison
   ```
   Enterキー

3. **次に：**
   ```
   npm run dev
   ```
   Enterキー

4. **「✓ Ready」というメッセージが表示されるまで待つ（30秒〜1分）**

5. **このメッセージが表示されたら、開発サーバーは起動しています**

---

### ステップ2: ブラウザで正しいURLを開く

1. **ブラウザを開く**

2. **アドレスバー（URLを入力するところ）をクリック**

3. **以下の文字を**正確に**入力：**
   ```
   http://localhost:3000
   ```

   ⚠️ 注意：
   - `http://` から始まる（`https://`ではない）
   - `localhost` と入力（`local`ではない）
   - ポート番号は `3000`
   - スペースは入れない

4. **Enterキーを押す**

---

### ステップ3: ポート番号を確認する

開発サーバーが別のポートで起動している可能性があります。

#### 確認方法：

コマンドプロンプトのウィンドウに、以下のような表示があるはずです：
```
○ Local:        http://localhost:3000
```

この数字（`3000`）がポート番号です。

もし、`http://localhost:3001` など、別の数字が表示されている場合は、その数字を使ってください。

例えば：
- `http://localhost:3001` と表示されていれば、ブラウザで `http://localhost:3001` を開く

---

## 🔍 トラブルシューティング

### Q1: コマンドプロンプトにエラーメッセージが表示されている

**確認すること：**

1. **エラーメッセージの内容を確認**
   - ターミナルに赤いエラーメッセージが表示されていないか確認
   - エラーメッセージを読んで、何が問題か確認

2. **よくあるエラー：**

   - **「npm が見つかりません」**
     → Node.jsがインストールされていない
     → Node.jsをインストールする必要があります

   - **「Port 3000 is already in use」**
     → ポート3000が既に使用されている
     → 解決方法：
       1. 別のポートを使用：
          ```
          npm run dev -- -p 3001
          ```
          その後、`http://localhost:3001` でアクセス
       2. または、ポート3000を使用しているプロセスを終了

   - **「Cannot find module」**
     → 依存関係がインストールされていない
     → 解決方法：
        ```
        npm install
        ```
        その後、もう一度 `npm run dev` を実行

### Q2: コマンドプロンプトに「✓ Ready」と表示されているが、ブラウザでアクセスできない

**確認すること：**

1. **URLが正しいか確認**
   - `http://localhost:3000` と入力しているか
   - スペルミスがないか
   - `localhost` が `local` になっていないか

2. **コマンドプロンプトのウィンドウが閉じられていないか確認**
   - 開発サーバーを起動したコマンドプロンプトのウィンドウは**開いたまま**にしておく必要があります
   - ウィンドウを閉じると、サーバーも停止してしまいます

3. **別のブラウザで試す**
   - Chromeで試す
   - Edgeで試す
   - Firefoxで試す

4. **ブラウザのキャッシュをクリア**
   - ブラウザの設定から、キャッシュをクリア
   - または、シークレットモード（プライベートモード）で開く

### Q3: ファイアウォールがブロックしている可能性

Windowsファイアウォールがブロックしている可能性があります。

**確認方法：**

1. **コマンドプロンプトで以下を実行：**
   ```
   netstat -ano | findstr :3000
   ```
   Enterキー

2. **何も表示されない場合**
   → サーバーが起動していない可能性があります

3. **表示された場合**
   → サーバーは起動しています
   → ブラウザの問題か、URLの問題の可能性があります

---

## 📋 確認チェックリスト

以下を順番に確認してください：

- [ ] 開発サーバーが起動している（コマンドプロンプトに「✓ Ready」が表示されている）
- [ ] コマンドプロンプトのウィンドウが閉じられていない
- [ ] ブラウザのアドレスバーに `http://localhost:3000` が入力されている
- [ ] URLにスペルミスがない（`localhost`が`local`になっていない）
- [ ] ポート番号が正しい（3000、または別の数字）
- [ ] 別のブラウザで試した

1. **コマンドプロンプトを開く**
   - Windowsキー → 「cmd」→ Enter

2. **プロジェクトフォルダに移動**
   ```
   cd C:\Users\kiris\.cursor\study-abroad-comparison
   ```
   Enterキー

3. **開発サーバーを起動**
   ```
   npm run dev
   ```
   Enterキー

4. **「✓ Ready」が表示されるまで待つ（30秒〜1分）**

5. **ブラウザで `http://localhost:3000` を開く**

6. **ページが表示されれば成功！**

## 💡 まだ解決しない場合

以下の情報を教えてください：

1. **コマンドプロンプトに何が表示されていますか？**
   - 「✓ Ready」が表示されているか
   - エラーメッセージが表示されているか
   - エラーメッセージの内容は何か

2. **ブラウザでどうなりますか？**
   - 「このサイトにアクセスできません」と表示される
   - 白い画面が表示される
   - その他のエラーメッセージが表示される

3. **どのURLを入力しましたか？**
   - `http://localhost:3000` か
   - 他のURLか

これらの情報があれば、より具体的に解決できます！
