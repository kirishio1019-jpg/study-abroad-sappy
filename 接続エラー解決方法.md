# 🔴 Connection Failed エラーを解決する方法

## 今すぐ試すこと

### 1. 開発サーバーが起動しているか確認

Cursorの**下部**（ターミナル）を見てください。

以下のようなメッセージが表示されていれば、サーバーは起動しています：
```
✓ Ready in X seconds
○ Local:        http://localhost:3000
```

**もしこのメッセージが表示されていない場合 → サーバーを起動する必要があります**

---

### 2. サーバーを起動する方法

1. **Cursorで `Ctrl + Shift + ` を押してターミナルを開く**
   （最後の` `は、数字の1の左にあるキー）

2. **ターミナルに以下を貼り付けてEnter：**
   ```
   cd C:\Users\kiris\.cursor\study-abroad-comparison
   ```

3. **次に、以下を貼り付けてEnter：**
   ```
   npm run dev
   ```

4. **30秒〜1分待つ**

5. **「✓ Ready」というメッセージが表示されるまで待つ**

---

### 3. ブラウザで開く

1. **ブラウザを開く**

2. **アドレスバー（URLを入力するところ）に以下を入力：**
   ```
   http://localhost:3000
   ```

3. **Enterキーを押す**

4. **ページが表示されれば成功！**

---

## ❓ よくある問題

### 問題1: 「npm が見つかりません」というエラー

**解決方法：**
- Node.jsがインストールされていない可能性があります
- まず、Node.jsをインストールする必要があります

### 問題2: サーバーは起動しているが、ブラウザで「接続できません」

**確認すること：**
1. ブラウザのアドレスバーに `http://localhost:3000` と入力されているか
2. `localhost` のスペルが正しいか（`localhost`で、`local`ではない）
3. `http://` から始まっているか（`https://`ではない）

### 問題3: ページは表示されるが、ログイン機能でエラーが出る

**解決方法：**
- Supabaseの環境変数が設定されていない可能性があります
- `.env.local`ファイルを作成して、Supabaseの認証情報を設定する必要があります
- 詳しくは `CONNECTION_FAILED_FIX.md` を参照してください

---

## 📞 まだ解決しない場合

以下の情報を教えてください：

1. **Cursorのターミナルに何が表示されていますか？**
   - 「✓ Ready」というメッセージが表示されているか
   - エラーメッセージが表示されているか

2. **ブラウザで `http://localhost:3000` を開いたとき、どうなりますか？**
   - 何も表示されない
   - 「接続できません」と表示される
   - エラーページが表示される

3. **どのようなエラーメッセージが表示されますか？**
   - ターミナルのエラーメッセージ
   - ブラウザのエラーメッセージ

これらの情報があれば、もっと具体的に解決できます！
